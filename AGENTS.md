# QFL Agents

## Versioning

- `+0.0.1` — small changes, features, hotfixes
- `+0.1.0` — top-level / major changes

## Git Rules

- Tag every push with `v{major}.{minor}.{patch}`
- No `.md` files except `README.md` — plans/docs must not leak to remote
- Run `cargo build` + `cargo test` before committing
